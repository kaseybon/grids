Layout Grid

Thoughts

I like how there are two separate grids, one for the layout and one for content. These are two different elements that need to be treated as such. In order to make the grids responsive you need to add classes that specify how many columns display for each break point. This just seems like too many potentially unused classes in your CSS. There is also the issue of having to create your own custom styles to override theirs if their break points won't work for you.

Block Grid:
I don't really like that you can only use the block grid with a list element. While this will probably be fine for most websites it's just a bit too restrictive when it doesn't have to be.

Layout Grid:
The layout grid is a 12 column grid that allows you to nest elements. Just like the Block grid classes are added depending on how many columns you want a container to span at a certain break point. If there are no classes the container will span all 12 columns. Other than the issues with having a bunch of classes I don't dislike this grid.
